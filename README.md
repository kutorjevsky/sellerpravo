# Seller Pravo — система управления выездными поручениями

Telegram-бот + Google-таблица (реестр) для юридической фирмы. Управляет поручениями
на выезды в суды, МФЦ, ГИБДД, к приставам и т.п.: создание, распределение, контроль
исполнения, отчёты, уведомления и поиск.

**Принцип:** каждое поручение — отдельная карточка с ID, статусом, сроком, исполнителем
и доказательством результата. Единый источник правды — Google-таблица; бот пишет в неё
через Google Sheets API.

## Возможности

- **Представитель**: создаёт поручение (мастер по шагам), видит свои, редактирует,
  отменяет, принимает результат (приёмка).
- **Секретарь**: видит все заявки, назначает исполнителя (с авто-подсказкой:
  курьер ежедневно / помощники Пн и Чт), ставит план-дату, меняет статусы.
- **Курьер/Помощник**: получает свои поручения, отчитывается о результате
  (текст + ссылка на фото/скан), отмечает повторный визит / невыполнение, комментирует.
- **Все**: поиск по всем поручениям; уведомления о событиях.

## Статусы поручения

`Новая заявка → Готово к маршрутизации → Назначено → Исполнено, ждёт приёмки → Принято`
плюс `Требуется уточнение`, `Повторный визит`, `Не исполнено объективно`,
`Просрочено`, `Отменено`.

## Стек

- Python 3.11, [aiogram 3](https://docs.aiogram.dev) (Telegram)
- [gspread](https://docs.gspread.org) + Google Sheets API (хранилище)
- Деплой: Docker / Railway

## Структура

```
bot/
├── main.py            # запуск
├── config.py          # роли, статусы, типы органов, схема колонок реестра
├── sheets.py          # слой доступа к Google-таблице (реестр)
├── setup_sheet.py     # создание/оформление структуры таблицы (1 раз)
├── states.py          # состояния диалогов
├── keyboards.py       # клавиатуры
├── utils.py           # карточки, даты, уведомления
└── handlers/
    ├── common.py        # /start, регистрация, поиск, карточка, комментарии
    ├── representative.py # создание, мои поручения, правка, отмена, приёмка
    ├── secretary.py      # заявки, назначение, план-дата, статусы
    └── executor.py       # мои поручения, результат, повторный визит, отказ
```

## Локальный запуск

```bash
pip install -r requirements.txt
cp .env.example .env          # заполнить BOT_TOKEN, SPREADSHEET_ID, ключ
python -m bot.setup_sheet     # один раз: создать структуру таблицы
python -m bot.main            # запустить бота
```
