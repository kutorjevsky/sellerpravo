# Внедрение системы управления поручениями

Пошаговая инструкция «с нуля». Не требует навыков программирования — нужно лишь
внимательно повторить шаги. Время на всё — около 1 часа.

---

## Что вы получите

```
┌──────────────┐     создаёт      ┌─────────────────────┐
│ Представитель│ ───поручение───▶ │   ТЕЛЕГРАМ-БОТ       │
└──────────────┘                  │  (удобный интерфейс) │
┌──────────────┐  назначает,      │                      │
│  Секретарь   │ ◀──видит всё────▶│   пишет напрямую в   │
└──────────────┘                  │          ▼           │
┌──────────────┐  отчёт,          │  ┌────────────────┐  │
│ Курьер/Помощ.│ ───комментарии──▶│  │ GOOGLE-ТАБЛИЦА │  │
└──────────────┘                  │  │    (РЕЕСТР)    │  │
                                  │  │ единый источник│  │
   уведомления всем ◀─────────────│  │    правды      │  │
                                  │  └────────────────┘  │
                                  └─────────────────────┘
```

- **Реестр** — это обычная Google-таблица, которую вы можете открыть в любой момент.
  Бот пишет в неё автоматически. Каждое поручение — отдельная строка (карточка)
  с ID, статусом, сроком, исполнителем, результатом и доказательством.
- **Бот** — мобильный интерфейс: представители ставят задачи, секретарь распределяет,
  исполнители отчитываются, все получают уведомления. Поиск по всем поручениям.

### Листы в таблице (создаются автоматически)
| Лист | Назначение |
|------|-----------|
| `00_Панель` | сводка: сколько новых, назначено, просрочено, ждёт приёмки |
| `01_Реестр` | **ядро** — все поручения, одна строка = одно поручение |
| `Справочник_Органы` | суды, МФЦ, приставы, ГИБДД: адреса, режимы приёма |
| `Справочник_Сотрудники` | кто есть кто, роли, доверенности (заполняется при регистрации) |
| `Справочник_Клиенты` | клиенты и дела (по желанию) |
| `Журнал` | автоматический лог: кто, что и когда сделал |

> Лишние сущности из черновика (отдельные «Входящие заявки», «Маршрут», «Отчёты»,
> «Доказательства») **намеренно объединены в один реестр**: бот сам дисциплинирует
> ввод (заменяет Google-форму), а «маршрут на завтра» и «архив» — это просто фильтры
> реестра по дате/статусу. Меньше листов = меньше путаницы.

## Шаг 1. Создать Telegram-бота (5 минут)

1. В Телеграме откройте **@BotFather**.
2. Отправьте `/newbot`.
3. Введите название (например, `Seller Pravo Задания`) и логин (например,
   `sellerpravo_tasks_bot` — должен заканчиваться на `bot`).
4. BotFather пришлёт **токен** вида `123456789:AAH...`. **Сохраните его** — это
   `BOT_TOKEN`.

---

## Шаг 2. Создать Google-таблицу (2 минуты)

1. Зайдите на https://sheets.new (создастся пустая таблица).
2. Назовите её, например, «Реестр поручений».
3. Посмотрите на адрес в браузере:
   `https://docs.google.com/spreadsheets/d/`**`1AbC...XyZ`**`/edit`
   Длинная часть между `/d/` и `/edit` — это **`SPREADSHEET_ID`**. Сохраните.

---

## Шаг 3. Создать сервисный аккаунт Google (15 минут, делается один раз)

Это «робот-пользователь», от имени которого бот будет редактировать таблицу.

1. Откройте https://console.cloud.google.com/ (войдите в тот же Google-аккаунт).
2. Сверху создайте новый проект: **Select a project → New Project**, имя любое
   (`sellerpravo`), нажмите **Create**, дождитесь создания и выберите проект.
3. Включите Google Sheets API:
   - перейдите по https://console.cloud.google.com/apis/library/sheets.googleapis.com
   - нажмите **Enable**.
4. Создайте сервисный аккаунт:
   - https://console.cloud.google.com/iam-admin/serviceaccounts
   - **Create Service Account** → имя `bot` → **Create and Continue** → **Done**.
5. Создайте ключ:
   - кликните на созданный аккаунт → вкладка **Keys** → **Add Key → Create new key**
   - тип **JSON** → **Create**. Скачается файл `*.json`. **Это секрет, не публикуйте.**
6. Откройте этот JSON-файл блокнотом и найдите строку
   `"client_email": "bot@....iam.gserviceaccount.com"` — **скопируйте этот email**.

---

## Шаг 4. Дать боту доступ к таблице (1 минута)

1. Откройте вашу Google-таблицу (из шага 2).
2. Нажмите **Настройки доступа (Share)**.
3. Вставьте **email сервисного аккаунта** из шага 3.6.
4. Дайте права **Редактор (Editor)**, снимите галку «уведомить», **Готово**.

> Без этого шага бот не сможет писать в таблицу — это самая частая ошибка.

---

## Шаг 5. Запустить бота на сервере (Railway, 20 минут)

Бот должен работать круглосуточно. Бесплатный/дешёвый вариант — **Railway**.

1. Залейте код в GitHub (этот репозиторий уже готов).
2. Зайдите на https://railway.app, войдите через GitHub.
3. **New Project → Deploy from GitHub repo** → выберите репозиторий `sellerpravo`.
4. Railway увидит `Dockerfile` и начнёт сборку. Дождитесь.
5. Откройте сервис → вкладка **Variables** → добавьте переменные:

   | Переменная | Значение |
   |-----------|----------|
   | `BOT_TOKEN` | токен из шага 1 |
   | `SPREADSHEET_ID` | ID из шага 2 |
   | `GOOGLE_CREDENTIALS_JSON` | **всё содержимое** JSON-файла из шага 3.5 (открыть блокнотом, выделить всё, скопировать, вставить) |
   | `AUTO_SETUP` | `1` — **только для первого запуска** |

6. Railway перезапустит бот. В логах (вкладка **Deployments → Logs**) появится:
   `Подключено: …`, затем `Оформление завершено`.
7. Откройте Google-таблицу — там появились листы `00_Панель`, `01_Реестр` и справочники
   с выпадающими списками и цветными статусами. 🎉
8. **Удалите переменную `AUTO_SETUP`** (или поставьте `0`) и перезапустите сервис —
   чтобы оформление не накладывалось повторно.

> **Локальный запуск (для теста)**: установите Python 3.11, выполните
> `pip install -r requirements.txt`, создайте файл `.env` по образцу `.env.example`
> (положите JSON в файл `credentials.json` и укажите путь в `GOOGLE_CREDENTIALS_FILE`),
> затем `python -m bot.setup_sheet` (один раз) и `python -m bot.main`.

## Шаг 6. Зарегистрировать сотрудников (5 минут)

1. Каждый сотрудник открывает бота и нажимает **/start**.
2. Вводит имя и выбирает роль: **Представитель / Секретарь / Курьер / Помощник**.
3. Сотрудник появляется в листе `Справочник_Сотрудники`.

> Роли можно поправить прямо в таблице (колонка «Роль») — изменения подхватятся.
> В колонке «Доверенности» секретарь может отметить, у кого какие доверенности.

---

## Шаг 7. Наполнить справочник органов (по желанию, но полезно)

Откройте лист `Справочник_Органы`. Несколько строк уже есть (суды ПН/ЧТ, приставы,
ГИБДД). Добавьте свои суды, МФЦ, адреса, режимы приёма. При создании поручения
бот будет предлагать эти органы кнопками и подставлять адрес.

## Как пользоваться (шпаргалка по ролям)

**Представитель**
- `➕ Новое поручение` — мастер из нескольких шагов (клиент → орган → задача → срок…).
- `📋 Мои поручения` — список своих активных, открыть карточку, **редактировать**,
  **отменить**, **принять результат** (приёмка).
- `🔍 Поиск` — по всем поручениям.
- Получает уведомления: назначен исполнитель, исполнено, комментарии.

**Секретарь**
- `📥 Новые заявки` — поручения от представителей; **назначить исполнителя**
  (бот подсказывает: Пн/Чт — суды для помощников, иначе — курьер) и **план-дату**.
- `🗓 На завтра` — что запланировано (это и есть «маршрут на завтра»).
- `📋 В работе`, `🔍 Поиск`, смена любого статуса.

**Курьер / Помощник**
- `📋 Мои поручения` — что назначено; по каждому: **✅ Исполнено** (результат +
  ссылка на фото/скан), **🔁 Повторный визит**, **🚫 Не исполнено**, **💬 Комментарий**.
- Результат и комментарии автоматически уходят инициатору и секретарю.

## Перенос старых данных (по желанию)

Старый файл (лист на каждый день) переносить целиком не нужно. Варианты:
- **Ничего не переносить** — начать с чистого реестра (рекомендуется).
- **Сохранить архив** — лист `ИСПОЛНЕННОЕ` из старого файла можно скопировать в
  отдельный лист `99_Архив` в новой таблице (только для истории, бот его не трогает).
  Готовый файл-выгрузку для импорта можно подготовить отдельно (см. `tools/`).

## Частые вопросы

**Бот не пишет в таблицу / ошибка доступа.** Проверьте шаг 4 — таблица должна быть
расшарена на email сервисного аккаунта с правами «Редактор».

**Бот «уснул» / не отвечает.** Проверьте, что сервис на Railway запущен (логи).
Telegram-бот при опросе (polling) должен работать постоянно.

**Как сменить роль сотруднику.** В таблице `Справочник_Сотрудники`, колонка «Роль»,
или сам сотрудник: команда `/changerole`.

**Можно ли редактировать прямо в таблице.** Да. Таблица — источник правды. Но статусы
лучше менять через бота, чтобы приходили уведомления.

**Безопасность.** JSON-ключ и токен бота — секреты. Они не хранятся в коде (см.
`.gitignore`), только в переменных окружения Railway.
